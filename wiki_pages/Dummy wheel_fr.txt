Un wheel <i>dummy</i> est un wheel bidon, ou factice, qui remplace certains paquets Python populaires disponibles sur PyPI, mais qui fonctionnent comme des modules sur notre infrastructure. Ils sont tous étiquetés <tt>dummy</tt> avec une version locale, par exemple <code>pyarrow-23.0.1+dummy.computecanada-py3-none-any.whl</code>.

Voici quelques exemples :
* [[Arrow/fr#PyArrow|pyarrow (Arrow)]]
* [[OpenCV/fr#Interfaces_Python|opencv_python (OpenCV)]]
* [[MPI4py/fr|MPI4py]]

= Dépannage =
{{Note|Dans le cas ci-dessous, <tt>pyarrow</tt> sert d'exemple, mais le résultat est le même pour d'autres paquets.}}

L'erreur suivante se produit quand pip sélectionne un wheel dummy parce que pip ne lui trouve pas de correspondance.

<span id="This_is_a_normal_error_generated_by_this_dummy_wheel."></span>
== Erreur  <i>This is a normal error generated by this dummy wheel.</i> ==

Si la dépendance <tt>pyarrow</tt> n'est pas satisfaite, une erreur est générée lors de la sélection du wheel dummy. Ceci est normal.
Les causes possibles sont&nbsp;:
# le module Arrow n'a pas été chargé et donc pip n'a pas pu trouver pyarrow;
# la version d'Arrow chargée ne correspond pas à la version requise par le paquet dépendant;
# le module Arrow chargé n'est pas compatible avec la version de Python utilisée.

=== Module non chargé ===
Dans ce cas, désactivez l'environnement virtuel, chargez le module, réactivez l'environnement virtuel et relancez la commande <code>pip install</code>.

1. Désactivez votre environnement virtuel.
{{Command|test $VIRTUAL_ENV && deactivate}}

2. Chargez le module Arrow.
{{Command|module load gcc arrow/x.y.z python/x.y.z}} 
où <tt>x.y.z</tt> est la version requise.

3. Réactivez l'environnement virtuel précédent.
{{Command|source <env>/bin/activate}} 

4. Lancez <code>pip install</code> de nouveau.
{{Command|pip install <package>}}

=== Version du paquet/module ===
Le module Arrow chargé ne répond pas à l'exigence. Par exemple, <tt>(from pyarrow>=21.0.0->datasets)</tt> signifie que cette version du paquet d'ensembles de données exige une version de <tt>pyarrow</tt> supérieure ou égale à <tt>21.0.0</tt>.

Autrement dit, l'utilisation d'une version antérieure avec cette version des ensembles de données fera l'installation du wheel dummy.

Vous pouvez trouver la version requise dans la ligne <tt>pyarrow_noinstall</tt>.
{{Command|pip install --no-index datasets
|result=
...
Processing /cvmfs/soft.computecanada.ca/custom/python/wheelhouse/generic/pyarrow_noinstall-9999+dummy.computecanada.tar.gz (from pyarrow>=21.0.0->datasets)
}}

<span id="No_Python_bindings"></span>
=== Aucune interface (<i>binding</i>) Python === 

Le module chargé ne dispose peut-être pas les interfaces Python nécessaires.
Vous pouvez le vérifier rapidement avec
{{Command|python -c "import pyarrow"}}
Si la commande ne retourne rien, l'importation a réussi et le module fournit <tt>pyarrow</tt> pour le module Python utilisé.

Vous pouvez aussi vérifier si pip le trouve.
{{Command
|pip list {{!}} grep pyarrow
|result=
pyarrow    23.0.0
}}

Si <tt>pip list</tt> affiche une entrée, alors <tt>pyarrow</tt> est disponible et visible par <tt>pip</tt>. S'il n'y a pas d'entrée, <tt>pyarrow</tt> n'est pas disponible.
