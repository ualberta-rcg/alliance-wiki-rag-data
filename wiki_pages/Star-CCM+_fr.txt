[[Category:Software]]
[https://mdx.plm.automation.siemens.com/star-ccm-plus STAR-CCM+] est une suite logicielle de simulation utilisée dans plusieurs spécialités de génie. Elle permet la modélisation dans des domaines variés dont l'acoustique, la dynamique des fluides, le transfert thermique, la rhéologie, l'écoulement polyphasique, le flux de particules, la mécanique des solides, les fluides réactifs, l'électrochimie et l'électromagnétisme.

= Limites de la licence = 
Nous avons l'autorisation d'héberger les binaires STAR-CCM+ sur nos grappes. Pour utiliser le logiciel, vous devez acquérir une licence auprès de  [https://www.plm.automation.siemens.com/global/en/buy/ Siemens]. [https://community.sw.siemens.com/s/article/How-faculty-members-in-academic-institutions-can-get-access-to-Simcenter-STAR-CCM Deux options sont disponibles]. La plupart des groupes de recherche opteront pour une  [https://community.sw.siemens.com/s/question/0D54O00006FKu39SAD/licensing-how-power-on-demand-pod-licensing-for-starccm-works licence Power-on-Demand (PoD)], qui se connecte simplement à un serveur de licence distant et ne nécessite qu'une clé de licence. La seconde option est plus complexe et nécessite la configuration et la gestion d'un serveur de licence institutionnel hébergé localement, ainsi que l'achat d'un pack académique Simcenter STAR-CCM+. Votre serveur de licence STAR-CCM+ devra être reconfiguré. Vous devez demander à l'admnistrateur du serveur de licence de votre établissement d'ouvrir les '''deux''' ports fournisseurs (flexible et statique) pour qu'ils puissent être rejoints à partir des nœuds NAT (''Network Address Translation'') des grappes que vous voulez utiliser. Pour obtenir la liste des nœuds NAT à fournir à votre administrateur, écrivez au [[Technical support/fr|soutien technique]] en indiquant 1) le nom de la ou des grappes 2) le nom de l'hôte (FQDN) ou l'adresse IP publique du serveur de license STAR-CCM+ et les numéros des plorts flexoible et statique.

== Configurer votre compte ==
Afin de configurer votre compte pour utiliser un serveur de licence avec le module Star-CCM+, créez le fichier  <code>$HOME/.licenses/starccm.lic</code> comme suit&nbsp;: 
{{File|name=starccm.lic|contents=SERVER <server> ANY <flexport>
USE_SERVER}}
où <code>server</code> et <code>flexport</code> sont remplacés respectivement par le nom de l'hôte (ou l'adresse IP) et le port statique du fournisseur du serveur de licence. Il n'est pas nécessaire de définir manuellement <code>CDLMD_LICENSE_FILE</code> comme étant égal à <port>@<server> dans votre script slurm puisque cette variable est automatiquement configurée quand un module Star-CCM+ est chargé.

<span id="PoD_license_file"></span>
=== Fichier pour une licence PoD ===

Si vous avez acheté une licence POD, votre variable d'environnement <code>LM_PROJECT</code> doit être manuellement configurée comme étant égale à <I>YOUR CD-ADAPCO PROJECT ID</i> dans votre script slurm. De plus, le fichier <code>~/.licenses/starccm.lic</code> doit être configuré comme suit sur toutes les grappes.
{{File|name=starccm.lic|contents=SERVER flex.cd-adapco.com ANY 1999
USE_SERVER}}

<span id="Cluster_batch_job_submission"></span>
= Soumettre des tâches en lot sur nos grappes = 

Avant de soumettre des tâches sur une grappe, vous devez configurer un fichier <code>~/.licenses/starccm.lic</code> sur chaque grappe où vous exécuterez des tâches. Si vous disposez d'une licence PoD, les modifications de pare-feu requises ont déjà été effectuées sur toutes nos grappes. Cependant, si vous utilisez un serveur de licence institutionnel local, vous devrez soumettre une [[Technical support/fr|demande d'assistance au soutien technique]] pour demander la modification ponctuelle du pare-feu réseau entre la ou les grappes et votre serveur de licence local. Si vous rencontrez toujours des problèmes pour faire fonctionner la licence, essayez de supprimer ou de renommer le fichier <code>~/.flexlmrc</code> car les chemins de recherche et/ou les paramètres précédents du serveur de licence pourraient y être stockés. Notez que des fichiers de sortie de tâches déjà exécutées peuvent s'accumuler dans des répertoires cachés nommés <code>.star-version_number</code> et consommer ainsi votre quota. Ceux-ci peuvent être supprimés en exécutant périodiquement <code>rm -ri ~/.starccm*</code> et en répondant <i>oui</i> à l'affichage de l'invite.

<span id="Graphical_use"></span>
= Mode graphique =

Pour travailler en mode graphique, nous recommandons d'utiliser un système [[Nibi/fr#Accès_via_Open_OnDemand_(OOD)|OnDemand]] ou JupyterLab pour démarrer un bureau distant. En plus de configurer <code>~/.licenses/starccm.lic</code>, les groupes qui possèdent une licence PoD devraient aussi exécuter <code>export LM_PROJECT='22digit-PoD-License-Key'</code> avant <code>starccm+</code>, comme dans les exemples ci-dessous; selon le type de licence, il faut aussi ajouter d'autres options comme <b>-power</b>. La commande <code>module avail starccm-mixed</code> affiche les versions de starccm qui peuvent être chargées dans l'environnement standard (StdEnv) qui est en place (2020 ou 2023). Autrement, la commande <code>module spider starccm-mixed</code> affiche toutes les versions <i>mixed</i> et <i>R8</i>  des modules qui peuvent être chargés dans les deux versions des modules d'environnement variable qui peuvent être utilisées (2020 ou 2023).

== JupyterLab ==
1. Démarrez une session JupyterHub Desktop en cliquant sur l'un des liens JupyterHub suivants :<br>
FIR : <code>https://jupyterhub.fir.alliancecan.ca</code>
NARVAL : <code>https://portail.narval.calculquebec.ca/</code>
RORQUAL : <code>https://jupyterhub.rorqual.alliancecan.ca</code><br>
2. Cliquez sur l'icône d'engrenage des modules logiciels située en bas du menu de sélection vertical le plus à gauche.<br>
3. Sélectionnez un module StarCCM tel que <code>starccm-mixed/20.06.010</code> **OU** <code>starccm/20.06.010-R8</code> et cliquez sur ''Load''.<br>
4. Sélectionnez <code>StarCCM+ Mixed (VNC)</code> **OU** Icône StarCCM (VNC) apparaissant sur le bureau.<br>
5. Pour exécuter StarCCM+ avec plusieurs cœurs&nbsp;:
: Cliquez sur ''File->New''. Le panneau de configuration <i>Create a File</i> devrait apparaître.
: Modifiez ''Serial Process Option''  en cliquant sur le bouton ''<b>Parallel on Local Host</b>''.
: Ajoutez <code>-mpi openmpi40</code> à la fin de la chaîne ''<b>Command</b>' située dans le bas du panneau.
: Enfin, cliquez sur le bouton <b>OK</b>. L’interface graphique de StarCCM+ devrait s’afficher.
