[https://arrow.apache.org/ Apache Arrow] est une plateforme de développement multilangage pour la gestion des données en mémoire. Elle utilise un format standardisé en colonnes qui organise les données hiérarchiques ou autres afin de permettre des opérations analytiques efficaces. La plateforme offre des bibliothèques de calcul, la transmission sans copie et en continu des données et la communication interprocessus. Parmi les langages pris en charge, on compte C, C++, C#, Go, Java, JavaScript, MATLAB, Python, R, Ruby et Rust.

== CUDA ==
Arrow est aussi disponible avec CUDA.
{{Command|module load gcc arrow/X.Y.Z cuda}}
où X.Y.Z désigne la version.

== ''Bindings'' Python ==
Le module contient des ''bindings'' pour plusieurs versions de Python. Pour connaître les versions compatibles, lancez
{{Command|module spider arrow/X.Y.Z}}
où X.Y.Z désigne la version.

Ou cherchez <i>pyarrow</i> directement avec
{{Command|module spider pyarrow}}

=== PyArrow ===
Les ''bindings'' Python (appelés ''PyArrow'') s’intègrent avec les objets de première classe NumPy, Pandas, et les objets natifs Python. Ils sont basés sur l'implémentation C++ de Arrow.

1. Chargez les modules requis.
{{Command|module load gcc arrow/X.Y.Z python/3.11}}
où X.Y.Z désigne la version.

2. Importez PyArrow.
{{Command|python -c "import pyarrow"}}

L’importation est réussie si rien n’est affiché.

Pour plus d'information, consultez [https://arrow.apache.org/docs/python/ la documentation Python].

==== Autres paquets Python dépendants ====
L'installation de certains paquets Python est dépendante de PyArrow.
Une fois le module <code>arrow</code> chargé, la dépendance à <code>pyarrow</code> sera satisfaite.
{{Command
|pip list {{!}} grep pyarrow
|result=
pyarrow    17.0.0
}}

If <tt>pip list</tt> show an entry, then <tt>pyarrow</tt> is available and seen by <tt>pip</tt>. Otherwise, in case of no entry, <tt>pyarrow</tt> is not available.

==== Format Apache Parquet ====
Le format de fichier [http://parquet.apache.org/ Parquet] est disponible. 

Pour importer le module Parquet, effectuez les étapes pour <code>pyarrow</code> ci-dessus et lancez ensuite
{{Command|python -c "import pyarrow.parquet"}}

L’importation est réussie si rien n’est affiché.

== ''Bindings'' R ==
Arrow possède une interface avec la bibliothèque Arrow C++ pour permettre l'accès en R de plusieurs de ses fonctionnalités. Ceci inclut l’analyse de grands ensembles de données multifichiers ([https://arrow.apache.org/docs/r/reference/open_dataset.html open_dataset()]); la capacité de travailler avec des fichiers individuels de format Parquet ([https://arrow.apache.org/docs/r/reference/read_parquet.html read_parquet()], [https://arrow.apache.org/docs/r/reference/write_parquet.html write_parquet()]) et Feather  ([https://arrow.apache.org/docs/r/reference/read_feather.html read_feather()], [https://arrow.apache.org/docs/r/reference/write_feather.html write_feather()]); l'accès à la mémoire et aux messages Arrow.

=== Installation ===
1. Chargez les modules requis.
{{Command|module load StdEnv/2020 gcc/9.3.0 arrow/8 r/4.1 boost/1.72.0}}

2. Spécifiez le répertoire d’installation local.
{{Commands
|mkdir -p ~/.local/R/$EBVERSIONR/
|export R_LIBS{{=}}~/.local/R/$EBVERSIONR/
}}

3. Exportez les variables requises pour vous assurer d’utiliser l'installation du système. 
{{Commands
|export PKG_CONFIG_PATH{{=}}$EBROOTARROW/lib/pkgconfig
|export INCLUDE_DIR{{=}}$EBROOTARROW/include
|export LIB_DIR{{=}}$EBROOTARROW/lib
}}

4. Installez les ''bindings''.
{{Command|R -e 'install.packages("arrow", repos{{=}}"https://cloud.r-project.org/")'}}

=== Utilisation ===
Une fois les ''bindings'' installés, il faut les charger.

1. Chargez les modules requis.
{{Command|module load StdEnv/2020 gcc/9.3.0 arrow/8 r/4.1}}

2. Chargez la bibliothèque.
{{Command
|R -e "library(arrow)"
|result=
> library("arrow")
Attaching package: ‘arrow’
}}

Pour plus d'information, consultez la [https://arrow.apache.org/docs/r/index.html documentation Arrow sur R].

= Troubleshooting = 

== This is a normal error generated by this dummy wheel. ==

When <tt>pyarrow</tt> dependency is not met, an error will be raised when the dummy wheel is selected. '''This is normal'''.
This means that either:
# the arrow module was not loaded, hence pip could not find pyarrow
# the arrow version loaded does not meet the requested version from the dependent package
# the arrow module loaded does not support the python version used

=== Arrow module not loaded ===
In this case, deactivate any virtual environment, load the module, re-activate the virtual environment, and re-run your pip install command.

1. Deactivate your virtual environment : 
{{Command|test $VIRTUAL_ENV && deactivate}}

2. Load the Arrow module :
{{Command|module load gcc arrow/x.y.z python/x.y.z}} 
where <tt>x.y.z</tt> is the version needed.

3. Re-activate your previously created virtual env. :
{{Command|source <env>/bin/activate}} 

4. And re-run your pip install command.
{{Command|pip install <package>}}

=== Arrow version ===
The arrow module loaded does not meet the requirement. For instance, <tt>(from pyarrow>=21.0.0->datasets)</tt> which means that this version of the datasets package requires <tt>pyarrow</tt> greater or equal to version <tt>21.0.0</tt>.

In other words, using any prior version with this version of datasets will results in the dummy wheel being installed.

You can find the asked requirement (version) by looking at the line (<tt>pyarrow_noinstall</tt>)
{{Command|pip install --no-index datasets
|result=
...
Processing /cvmfs/soft.computecanada.ca/custom/python/wheelhouse/generic/pyarrow_noinstall-9999+dummy.computecanada.tar.gz (from pyarrow>=21.0.0->datasets)
}}

=== No Python bindings ===

The loaded module may not have the python bindings for the python module loaded.
You can quickly verify by using:
{{Command|python -c "import pyarrow"}}
If the command displays nothing, the import was successful and the module provide <tt>pyarrow</tt> for the python module used.

Or one can check if pip sees it as demonstrated in [[#Fulfilling other Python package dependency]]
{{Command
|pip list {{!}} grep pyarrow
|result=
pyarrow    23.0.0
}}

If <tt>pip list</tt> show an entry, then <tt>pyarrow</tt> is available and seen by <tt>pip</tt>. Otherwise, in case of no entry, <tt>pyarrow</tt> is not available.
