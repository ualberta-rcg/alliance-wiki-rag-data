Dummy wheels exist in the wheelhouse to provide placeholders for some popular Python packages that are available on PyPI, but are modules on our infrastructure. They are all tagged (<tt>dummy</tt>) with a local version, for instance  <code>pyarrow-23.0.1+dummy.computecanada-py3-none-any.whl</code>.

Such examples are:
* [[Arrow#PyArrow|pyarrow (Arrow)]]
* [[OpenCV#Python_bindings|opencv_python (OpenCV)]]
* [[MPI4py|MPI4py]]

= Troubleshooting =
{{Note|In the error message described below, <tt>pyarrow</tt> is used as an example, but it also occurs in other packages as well.}}

When a dummy wheel is selected by pip, an error is raised because pip could not find a match for this requirement.

== This is a normal error generated by this dummy wheel. ==

When a <tt>pyarrow</tt> dependency is not met, an error is raised when the dummy wheel is selected. '''This is normal'''.
Possible reasons are:
# the Arrow module was not loaded, hence pip could not find pyarrow;
# the Arrow version loaded does not meet the requested version from the dependent package;
# the Arrow module loaded does not support the Python version used.

=== Module not loaded ===
In this case, deactivate the virtual environment, load the module, re-activate the virtual environment, and re-run the <code>pip install</code> command.

1. Deactivate your virtual environment. 
{{Command|test $VIRTUAL_ENV && deactivate}}

2. Load the Arrow module.
{{Command|module load gcc arrow/x.y.z python/x.y.z}} 
where <tt>x.y.z</tt> is the version needed.

3. Re-activate your previously created virtual environment.
{{Command|source <env>/bin/activate}} 

4. And re-run the <code>pip install</code> command.
{{Command|pip install <package>}}

=== Package/module version ===
The Arrow module loaded does not meet the requirement. For instance, <tt>(from pyarrow>=21.0.0->datasets)</tt> means that this version of the datasets package requires <tt>pyarrow</tt> greater or equal to version <tt>21.0.0</tt>.

In other words, using any prior version with this version of datasets will results in the dummy wheel being installed.

You can find the requirement (version) by looking at the line (<tt>pyarrow_noinstall</tt>)
{{Command|pip install --no-index datasets
|result=
...
Processing /cvmfs/soft.computecanada.ca/custom/python/wheelhouse/generic/pyarrow_noinstall-9999+dummy.computecanada.tar.gz (from pyarrow>=21.0.0->datasets)
}}

=== No Python bindings === 

The loaded module may not have the Python bindings for the Python module loaded.
You can quickly verify by using
{{Command|python -c "import pyarrow"}}
If the command displays nothing, the import was successful and the module provides <tt>pyarrow</tt> for the Python module used.

Or you can check if pip sees it.
{{Command
|pip list {{!}} grep pyarrow
|result=
pyarrow    23.0.0
}}

If <tt>pip list</tt> shows an entry, then <tt>pyarrow</tt> is available and seen by <tt>pip</tt>. Otherwise, if there is no entry, <tt>pyarrow</tt> is not available.
